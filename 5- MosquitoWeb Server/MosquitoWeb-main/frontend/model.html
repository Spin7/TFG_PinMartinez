Modelo IA — Pipeline Cascada

Arquitectura de dos etapas: detección con YOLOv8s seguida de clasificación con MobileNetV3-Large, todo exportado a ONNX para inferencia embebida en Raspberry Pi 5.

Descripción General

Sistema en Cascada — Dos Modelos

El modelo utilizado es un modelo cascada compuesto por dos etapas secuenciales. La primera etapa está basada en YOLOv8s (versión Small) y se encarga de la detección de mosquitos en la imagen completa, generando bounding boxes sobre cada instancia detectada.

La segunda etapa, basada en MobileNetV3-Large, recibe los recortes (crops) de cada detección y realiza la clasificación binaria entre machos y hembras. Este diseño en dos etapas permite mantener alta sensibilidad en la detección sin sacrificar la precisión en la clasificación de género.

Todo el sistema ha sido convertido al formato ONNX para su despliegue en dispositivos Edge (Raspberry Pi 5) utilizando el framework ONNX Runtime, logrando una reducción de ~2× en tiempo de inferencia respecto al modelo PyTorch nativo ejecutado en CPU.

Etapa 1

YOLOv8s — Detector de Mosquitos

Descripción

¿En qué destaca YOLOv8s?

YOLO (You Only Look Once) es la familia de detectores de objetos en tiempo real más utilizada en visión por computadora. Procesa la imagen completa en un único pase de red, prediciendo simultáneamente bounding boxes y puntuaciones de clase sin anclas predefinidas. La versión YOLOv8, de Ultralytics, incorpora una cabeza de detección anchor-free, funciones de pérdida CIoU/DFL para ajuste de caja más preciso, y un backbone CSP-Darknet mejorado con conexiones C2f.

Sus puntos fuertes para este proyecto son: alta velocidad de inferencia incluso en CPUs embebidas, escalabilidad (variantes n/s/m/l/x), soporte nativo a exportación ONNX/TensorRT y la comunidad de Ultralytics con datasets y herramientas de augmentación integradas. Se eligió la variante YOLOv8s (Small) con entrada 256×256 px y entrenamiento con una única clase (Mosquito) para maximizar el throughput en Raspberry Pi 5.

Etapa 2

MobileNetV3-Large — Clasificador de Género

Descripción

¿En qué destaca MobileNetV3-Large?

MobileNetV3 es una familia de redes convolucionales ligeras optimizada para inferencia eficiente en dispositivos móviles y embebidos. Combina bloques Inverted Residuals con activación h-swish, mecanismos de atención por canal (Squeeze-and-Excitation) y fue diseñada mediante Neural Architecture Search (NAS) para lograr el mejor balance accuracy/latencia.

Sus puntos fuertes para este proyecto son: muy bajo número de parámetros (~5.5 M), alta accuracy en clasificación binaria con imágenes de 160×160 px, y exportación directa a ONNX con ejes dinámicos de batch. Se utilizó la variante MobileNetV3-Large con el clasificador final reemplazado por una capa lineal de 2 salidas (Female / Male), entrenada sobre crops de mosquitos generados por el detector YOLO.

MobileNetV3-Large h-swish activation SE Attention blocks Clases: Female / Male 160×160 px input ~5.5 M parámetros
